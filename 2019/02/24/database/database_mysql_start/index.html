MySQL编译调试方法介绍

MySQL编译调试

先卸载安装的mysql

systemctl stop mysql
sudo apt-get autoremove –purge mysql-server-5.7

源码编译

初始化MySQL实例

方法一：使用默认参数初始化实例

查看默认的配置参数：

./mysqld --verbose --help

初始化MySQL数据库实例

./mysqld --initialize

方法二：通过命令行指定默认参数

./mysqld –initialize –character-set-server=UTF8MB4 \
–basedir=/home/mysql/work/mysql-8.0.14/release/ \
–datadir=/home/mysql/work/mysql-8.0.14/release/data \
–lc-messages-dir=/home/mysql/work/mysql-8.0.14/release/ \
–log-error=/home/mysql/work/mysql-8.0.14/release/error.log \
–user=mysql

方法三：通过配置文件指定默认参数

MySQL读取配置文件的优先级如下：

/etc/my.cnf /etc/mysql/my.cnf /usr/etc/my.cnf ~/.my.cnf

在bin目录下创建文件my.cnf

初始化MySQL数据库实例

./mysqld --initialize

启动MySQL实例

./mysqld --debug --console &

启动MySQL客户端

./mysql -uroot -pitn/rv5dwTEw  #初始化的error.log会显示默认的root用户密码

调试MySQL常用断点

handle_query #查询先关
create_table
Sql_cmd_insert_values
mysql_execute_command

SQL查询相关

mysql_select #执行查询操作
mysql_insert #执行insert操作
mysql_update #执行update操作
mysql_delete #执行delete或truncate操作

网络接口

my_net_read      #从客户端获得网络数据包
my_net_write     #向客户端发送网络数据包
check_connection #网络连接认证

复制功能

Start_slave_threads #从服务器进程启动
handle_slave_io     #从服务器的网络读写进程
handle_slave_sql    #从服务器的SQL执行进程

存储引擎

openfrm #打开表定义文件frm
open_table #打开一个表
Mysql_lock_tables #获得表锁
MYSQL_LOCK::write(Log_event*) #记录二进制更新日志
