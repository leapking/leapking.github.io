Linux动态库链接的那些事儿

发表于 2018-03-10 | 分类于 linux ， lib
Linux动态库链接使用的介绍

Glibc安装的库中有一个名为ld-linux.so.*的非常重要的库，它负责动态链接库的查找。每一个使用动态库的程序都会依赖于它。

在Linux上动态链接库的查找路径又分两种情况：

  • 链接时路径（Link-time path）
  • 运行时路径（Run-time path）
    动态库的链接和运行时的目录是分开指定的。

链接时路径（Link-time path）

编译时，ld-linux.so查找共享库的顺序：GCC_EXEC_PREFIX -> LIBRARY_PATH -> ld -> include

  1. ld-linux.so根据gcc –print-search-dirs所打印出的路径查找一些动态库，主要是libgcc_s.so等库。可以通过GCC_EXEC_PREFIX来设定。
    ld-linux.so.6由gcc的spec文件中所设定。
  2. LIBRARY_PATH环境变量中所设定的路径，或编译的命令行中指定的-L/usr/local/lib。
  3. binutils中的ld所设定的缺省搜索路径顺序（编译binutils时指定），可以通过“ld –verbose | grep SEARCH”来查看。

运行时路径（Run-time path）

运行时，ld-linux.so查找共享库的顺序：DT_RPATH -> LD_LIBRARY_PATH -> ld.so.conf -> /lib和/usr/lib

  1. ld-linux.so.6根据可执行程序的.dynamic 段的DT_RPATH来查找库，DT_RPATH是在编译时由gcc的-wl,rpath或-R设置的。
    example: gcc -Wl,-rpath,/home/arc/test,-rpath,/lib/,-rpath,/usr/lib/,-rpath,/usr/local/lib test.c
    ld-linux.so.6的路径在可执行的目标文件中被写死，可用readelf命令查看。

  2. LD_LIBRARY_PATH环境变量中所设定的路径

  3. /etc/ld.so.conf（或/usr/local/etc/ld.so.conf）中所指定的路径，由ldconfig生成二进制的ld.so.cache中（修改/etc/ld.so.conf需重新执行ldconfig）。
  4. 查找默认路径/lib和/usr/lib

链接与运行时动态库查找顺序的比较

  1. 编译时查找的是静态库或动态库，而运行时，查找的只是动态库。
  2. 编译时可以用-L指定查找路径，或者用环境变量LIBRARY_PATH，而运行时可以用-Wl,rpath或-R选项，或者修改/etc/ld.so.conf文件或者设置环境变量LD_LIBRARY_PATH.
  3. 编译时用的链接器是ld，而运行时用的链接器是/lib/ld-linux.so.2.
  4. 编译时与运行时都会查找默认路径：/lib /usr/lib
  5. 编译时还有一个默认路径：/usr/local/lib，而运行时不会默认找查该路径。
